## Goal
- Complete web platform ("Fantasy Mundial") for daily football predictions during FIFA World Cup 2026 with payments, automated scoring, weekly prizes, admin panel, and polished pink/magenta dark theme.

## Constraints & Preferences
- **Payments**: $2,000 COP/day via **Wompi** production API (`https://api.wompi.co/v1/payment_links`).
- **Database**: **MySQL** on Railway (`mysql.railway.internal:3306`, user: `root`, db: `railway`). Local MySQL80 on `localhost:3306` (pass: `1118023359AAPV`).
- **Sports APIs**: **Bzzoiro Sports Data** (`https://sports.bzzoiro.com/api/v2/`, token `261c45543020e4f0919e2796417bf27744005aef`, league_id=27, **season_id=188** for WC2026) + **worldcup2026.ir** (free, all 104 matches). Both used.
- **Auth**: Pure password-based (WhatsApp + password) via NextAuth v5 (`next-auth@5.0.0-beta.31`).
- **Prediction fields**: 13 fields, max 42 pts/match. Extra predictions +$2,000 via Wompi.
- **Prizes**: Configurable per position (1st-10th) per week via admin panel (`WeekPrize` model). **Auto-assigned** when week closes.
- **Prize codes**: Format `FA` + 4 random digits (e.g., `FA3821`). Valid for **5 days** after generation.
- **Admin panel**: Password auth via `ADMIN_PASSWORD`, generate/manage API Keys, configure prizes with image upload (Cloudinary), validate/redeem codes, view winners/partidos/usuarios.
- **Auto-sync**: via `instrumentation.ts` cron (3 min recursive setTimeout) + `syncLiveMatchesIfNecessary()` on every API request (15s cooldown).
- **Live updates**: Dashboard polls every 15 seconds. Match detail polls every 15 seconds when live. `syncLiveMatchesIfNecessary()` fetches from Bzzoiro API, updates status/scores/stats in DB.
- **Design**: Dark theme with hot pink/magenta accents (#FF00FF, #FF1493), 9 images from local folder as per-page backgrounds.
- **All dates in Colombia timezone (UTC-5)**: Matches, DayPasses, dashboard grouping, display. API dates converted via `parseEventDate()` + `toColombiaDate()`.
- **Weekly flow**: Lun-Dom. Scores reset each week. Closed week auto-assigns prizes. Users see their prizes in `/mis-premios`. Ranking page has historical week selector.
- **GitHub repo**: `Joker8-h/polla-mundialista` — connected to Railway for auto-deploy on push.

## Key Decisions
- **`closeWeekAndAssignPrizes()` helper**: Lógica compartida entre instrumentation.ts y cron/sync. Genera códigos FA únicos con retry loop (20 intentos).
- **Codes FA + 4 dígitos**: Formato fácil de recordar para tienda física. Garantizado único vía `findUnique`.
- **expiresAt = 5 días**: Tiempo suficiente para canjear en tienda física.
- **weekPrizeId en WeeklyWinner**: Relación directa al premio configurado, así el ticket muestra label + valor + imagen.
- **Dashboard ahora usa `/api/matches/week/[weekId]`**: En vez del antiguo endpoint -2/+7 días. Muestra solo partidos Lun-Dom.
- **WeekPrize.winners[]**: Relación inversa agregada para queries eficientes.
- **`getWeekDays()`**: Genera array de días entre startDate y endDate. Filtra `dayKeys` a solo días con partidos.
- **Ranking histórico**: Selector de semanas con indicador visual de semana activa (punto rojo).
